# Systems, Not Targets

Targets are entry points. Systems are relationships.

We map **architecture and dependencies**:

- How do components connect?
- Where does trust flow between layers?
- What relationships exist outside public scope?
- Which boundaries create separate behavior zones?

Single endpoints reveal little.  
The topology between them reveals structure.

Reality exists in transitions:

- proxy → application
- frontend → backend
- public → internal
- expected → protected

Understanding a system means understanding how these transitions behave under interaction.

**Core Principle:** Systems reveal what targets conceal.
